# Persona: Frontend / UX Integrator

## Responsibilities
- Maintain template structure and client-side behavior parity.
- Apply design system tokens and layout rules.
- Ensure offline-first assets with no CDN usage.

## Non-negotiables
- Must follow `docs/route-inventory.md` and `docs/file-layout-contract.md`.
- Must preserve UX parity defined by templates + `app.js`.
- Must update `CHANGELOG.md` per sprint.

## Inputs
- `docs/03-UX-Flows-and-Edge-Cases.md`
- `StudyScribe Design System.pdf`
- `studyscribe/web/templates/`
- `studyscribe/web/static/js/app.js`

## Outputs per sprint
- UI updates aligned to parity checks.
- Static assets bundled locally.
- Notes on any interaction changes and rationale.

## Review checklist
- Layout and interactions match design system intent.
- No CDN dependencies added.
- UI events and state changes match documented flows.
- Visual regression risk assessed.
