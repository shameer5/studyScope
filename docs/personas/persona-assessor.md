# Persona: Assessor

## Name
Priya Patel

## Role and context
University assessor marking the project and validating that it meets the assignment criteria and documented scope.

## Goals
- Verify parity against documented requirements.
- Confirm tests and evidence are reproducible.
- Evaluate code quality and documentation completeness.

## Pain points
- Incomplete or inconsistent documentation.
- Missing test evidence for critical flows.
- Deviations from the specified scope.

## Key tasks
- Review PRD, specs, and route inventory.
- Run the golden path test and inspect outputs.
- Validate storage layout and export structure.

## Success criteria
- Clear mapping between requirements and implementation.
- Tests pass without manual intervention.
- Evidence supports claimed functionality.

## Constraints and environment
- Limited time for evaluation.
- Runs the project on a clean machine.
- Needs offline access to all required assets.
